# Tools/ 目录导读

这个目录包含 CPython 的维护、生成、benchmark 和开发支持工具。

旧的 `Tools/README` 仍然保留并且依然有用；这份文件则把它整理成了一张更高层的地图。

## 这里放了什么

`Tools/` 不是运行时本体；它是围绕运行时的一层支持系统。

典型内容包括：

- 代码生成器
- 解析器生成辅助工具
- 调试辅助工具
- 构建与打包支持
- demo 与 benchmark
- 独立维护脚本

## 重要子目录

- `peg_generator/` —— PEG 解析器开发相关的生成工具
- `gdb/` —— 在 gdb 中调试 CPython 的辅助工具
- `scripts/` —— 一些有用的独立维护脚本
- `freeze/` —— 冻结 Python 应用的工具
- `msi/` —— Windows 打包支持
- `demo/` —— 示例和演示

## 为什么这个目录重要

你通常会在想弄清“CPython 是如何构建、调试或重新生成某些产物的”时阅读 `Tools/`，而不是在想弄清字节码是如何在运行时执行时来这里。

例如：

- frozen importlib 的生成会同时依赖这里和 `Programs/` 中的工具
- PEG 解析器的演进依赖这里的生成工具
- 调试复杂解释器状态时，`gdb/` 往往很有帮助

## 这个目录如何连接到整棵源码树

- 支持 parser / importlib / runtime 相关产物的重新生成与维护
- 帮助开发者在修改 `Python/`、`Objects/` 和 `Modules/` 时检查运行时状态

## 推荐阅读顺序

1. 先读这份文件
2. 再读原始的 `Tools/README`
3. 然后进入与你当前任务最相关的工具子目录

- 哪些工具负责重新生成解析器相关产物？
- 调试和维护用的辅助脚本放在哪里？
- CPython 开发工作流里有哪些支持程序？
